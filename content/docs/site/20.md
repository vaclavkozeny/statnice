Principy vrstvené architektury počítačových sítí, referenční model OSI. Charakteristika lokálních počítačových sítí

## Vrstvená architektura
- rozdělení složitého procesu síťové komunikace do menších, logicky oddělených a snáze spravovatelných částí -> **vrstva**
- abstrakce a nezávislost 
	- vrstva neví jak fungují okolní vrstvy
	- slouží vrstvě nad a využívá vrstvu pod
- modularita
	- můžeme změnit technologii na jedné vrstvě a okolní to nepoznají (ETH/WIFI)
- zapouzdření (encapsulation)
	- vrstvy balí data do jednoho balíku
	- každá přidá svojí hlavičku a kontrolní informace
	- na opačné straně si každá vrstva odebírá potřebné informace v opačném pořadí
- vertikální komunikace pomocí **API**
- horizontální komunikace pomocí **protokolů**
## OSI/ISO
- abstraktním modelem reálného systému
- nespecifikuje implementaci systémů
- uvádí všeobecné principy architektury
- 7 vrstev
### L1
- physical layer 
- definuje elektrické a fyzikální vlastnosti
	- rozložení pinů
	- napěťové úrovně 
	- vlastnosti kabelů
- stanovuje způsob přenosu
- modulace signálu
#### Zařízení
- HUB
	- odesílá data na všechny porty
- Repeater
### L2
- link layer
- umožňuje komunikaci v LAN
- uspořádává data do rámců (**frame**)
- **MAC address** (fyzická adresa)
	- 01:23:45:67:89:ab
	- media access controll
	- MAC adresy -> spjaty s NIC
	- identifikátor síťového zařízení
	- broadcats se šíří až po router
- detekuje a opravuje chyby vzniklé na fyzické vrstvě
#### LLC
- IEEE 802.2
- spojuje link-local s síťovou
- řídí tok dat mezi síťovými zařízeními
- LLC se stará o obsah a formátování rámce
#### ARP
- address resolution protocol
- získání MAC adresy z IP adresy
#### Frame
- preambule (identifikuje začátek rámce) - střídající se 1 a 0
- SFD (start frame delimiter) - značí konec preambule
- hlavička (obsahuje zdrojovou a cílovou MAC adresu, délku)
- data (max 1500 oktetů)
- kontrolní posloupnost rámce (kontrolní součet dat) - 32bit CRC (cyklický redundantní součet)
- mezera mezi packety (časový interval, kdy nic nesmí vysílat - odděluje packety)
![Frame](/statnice/mac.jpg)
#### Zařízení
- Bridge
- Swich
	- posílá podle MAC adresy
	- když nemá adresu v CAM tabulce odešle na všecnhy porty (kromě vstupního portu)
	- **Store and forward**
	    - kontroluje CRC
	    - pomalejší
	    - různé přenosové rychlosti
	- **Cut-throught**
	    - data posílá rovnou po přečtení cílové MAC adresy
	    - rychlejší
### L3
- network layer
- protokoly pro směrování dat
	- RIP - distance vector
	- OSPF - link-state 
	- EIGRP - hybrid
	- BGP - distance vector
- Router posílá pakety do cizí sítě
	- volí nejlepší cestu
- data která nemůžeme doručit přímo doručíme:
	- tomu kdo je blíže k cíli (next-hop)
	- tomu kdo má větší šanci najít správnou cestu (default-route)
- **Routing table**
	- jak doručit data do neznámých sítí
	- adresa cílové sítě + masku
	- gateway sítě - název odchozího rozhraní
	- IP adresa dalšího routeru
	- metrika (výhodnost cesty)
- Media independent
- **IP adresy** 
	- v4
	- v6
#### Packet
- verze
- délka
- time to live
- adresy
- data
#### Zařízení
- **Router**
    - routovací tabulka
    - neznáme packety zahazuje
    - posílá packety do cílových síťí
    - výběr nejlepší cesty
- **L3 switch**
    - umí vše co L2 + router
    - routuje VLAN
    - rychlejší než router
### L4
- transport layer
- protokoly
	- TCP
		- spojovaný, spolehlivý, zaručuje doručení a pořadí, flow control, congestion control
	- UDP
		- nespojovaný, bez záruky doručení, rychlejší, vhodný pro streaming, DNS, VoIP
- **port number**
#### Segment TCP
- porty
- sequance number
- ack number
- flagy
- window size
#### Segment UDP
- porty 
- délka
### L5
- session layer
- spravuje komunikaci mezi dvěma aplikacemi
- Zakládá a udržuje relaci (session) mezi aplikacemi
- synchronizace dat
- NetBIOS
### L6
- Presentation layer
- kóduje, komprimuje, šifruje, hashuje
### L7
- vrstvu představují konkrétní aplikace
## TCP/IP
- vrstvený systém podobný OSI/ISO
- spojuje některé vrstvy dohromady 
- L1+L2 -> physical layer
- L3 -> Network layer
- L4 -> Transport
- L5+L6+L7 -> application layer

## LAN
- nízká latence
- soukromě vlastněná a spravovaná
- omezený dosah (typicky do ~1 km)
### Topologie
- **hvězda** – všechna zařízení připojena do centrálního switche
- **sběrnice** – historická, všichni sdílejí jedno médium, kolize řešeny CSMA/CD.
- **kruh** – data obíhají dokola, historická
### Média
- kroucená dvojlinka
- optické vlákno
- WiFi (IEEE 802.11)
