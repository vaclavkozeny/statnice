Webové aplikace – charakteristika programování na straně serveru a klienta. Základy jazyka JavaScript. DOM a přístup k prvkům stránky. 

## Webové aplikace

### Server-side
- kód běží na serveru
- klient dostane až výsledek (HTML stránku, JSON data)
- klient nevidí zdrojový kód

- přístup k databázi, souborovému systému, externím službám
- autentizace a autorizace - bezpečná, klient nemůže obejít
- generování HTML dynamicky podle dat (šablony)
- stavové - server si může pamatovat session
### Technologie
- Node.js
- Python (Django, Flask)
- PHP
- Java (Spring)
- C# (ASP.NET)
### Typy architektur
- **SSR (Server-Side Rendering)** - server vygeneruje celé HTML a pošle ho klientovi
- **REST API** - server vrací jen data (JSON), klient si z nich sám vykreslí stránku
- **GraphQL** - alternativa k REST, klient si přesně specifikuje jaká data chce

---

### Client-side
- kód běží v prohlížeči uživatele
- server pošle JS soubory a prohlížeč je vykoná

- manipulace s DOM (změna stránky bez reloadu)
- reakce na uživatelské akce (kliknutí, scroll, input)
- komunikace se serverem na pozadí (AJAX, fetch)
- nemá přístup k databázi ani souborovému systému (bezpečnostní sandbox)
### Technologie
- JavaScript (jediný nativní jazyk prohlížeče)
- TypeScript (kompiluje se do JS)
- frameworky React, Vue, Angular
### Architektura
- **CSR (Client-Side Rendering)** - server pošle prázdné HTML + JS, prohlížeč si sám vykreslí stránku
## JS
- dynamicky typovaný
- interpretovaný
- single-threaded
- multiplatformní
- asynchronní programování

- standardizace ECMAScript
### Proměnné
```js
var x = 1;      // starý způsob, vyhýbat se – function scope, hoisting
let y = 2;      // blokový scope, lze přeřadit
const z = 3;    // blokový scope, nelze přeřadit (ale objekt lze mutovat)
```
### Datové typy
- primitivní: `string`, `number`, `boolean`, `null`, `undefined`, `symbol`, `bigint`
- objektové: `object`, `array`, `function`
### Async
```js
// Callback – starý způsob, vede k "callback hell"
fetch(url, function(data) { ... });

// Promise
fetch(url)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await – moderní, čitelnější
async function getData() {
  try {
    const response = await fetch(url);
    const data = await response.json();
    return data;
  } catch (err) {
    console.error(err);
  }
}
```
### Reaktivní frameworky

- problém -> ruční synchronizace data s tím co je vidět na stránce
- řešení -> UI je automaticky funkcí stavu - změníš data, framework sám překreslí co je potřeba
**Hlavní frameworky:**
- **React** - knihovna od Meta, nejrozšířenější. Komponenty, JSX (HTML v JS), virtuální DOM. Jednosměrný tok dat.
- **Vue** - jednodušší na začátek, postupný adopce (lze přidat do existující stránky). Dvoucestný binding.
- **Angular** - kompletní framework od Google, TypeScript, přísnější struktura. Vhodný pro velké enterprise projekty.
**Virtuální DOM**
- React a Vue nemanipulují přímo s DOM 
- udržují si vlastní lehkou kopii stromu v paměti, porovnají co se změnilo (diffing) a do skutečného DOM zapíší jen rozdíly
## DOM
- stromová struktura aplikace
```
document
└── html
    ├── head
    │   └── title
    └── body
        ├── h1
        └── div
            └── p
```
### Přístup k prvkům
```js
// podle ID – vrátí jeden element
document.getElementById("hlavni")

// podle CSS selektoru – vrátí první shodu
document.querySelector(".trida")
document.querySelector("#id")
document.querySelector("div > p")

// podle CSS selektoru – vrátí všechny shody (NodeList)
document.querySelectorAll(".polozka")

// starší metody
document.getElementsByClassName("trida")  // HTMLCollection
document.getElementsByTagName("p")
```
- z JS se dají prvky modifikovat
### Události
```js
const btn = document.querySelector("button");

btn.addEventListener("click", function(event) {
  console.log("kliknuto");
  event.preventDefault(); // zabrání výchozí akci (submit formuláře, přechod odkazu)
});
```
