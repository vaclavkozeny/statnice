Jazyk PHP. Problematika uchovávání stavových informací, cookies.

## PHP
- **mrtvý jazyk**
- serverový skriptovací jazyk
- kód běží na serveru, klient dostane jen výsledné HTML

- interpretovaný, dynamicky typovaný
- kód se vkládá přímo do HTML pomocí `<?php ... ?>`
- soubory mají příponu `.php`
- široce rozšířený – pohání velkou část webu
## State
- HTTP je stateless protokol
- **Cookies** 
	- malé soubory uložené v prohlížeči, posílají se s každým requestem
	- atributy
		- `Expires` / `Max-Age` - jak dlouho platí
		- `Path` - pro jakou cestu na serveru se cookie posílá
		- `Domain` - pro jakou doménu platí
		- `Secure` - posílá se jen přes HTTPS
		- `HttpOnly` - JavaScript k ní nemá přístup (ochrana proti XSS krádeži cookie)
		- `SameSite` - ochrana proti CSRF útokům (`Strict`, `Lax`, `None`)
	- omezení
		- uživatel je může smazat nebo zakázat
		- max ~4 KB dat
		- viditelné uživateli
		- posílají se s každým requestem
- **Session**
	- stav uložen na serveru, klient má jen session ID (v cookie)
	- omezení:
		- při více serverech musí být session data sdílená
		- jinak uživatel při přesměrování na jiný server "ztratí" session
- **JWT token**
	- stav zakódován přímo v tokenu, server nic neukládá
	- server jen ověří podpis tokenu
- **localStorage / sessionStorage**
	- úložiště v prohlížeči přístupné přes JavaScript
	- větší kapacita než cookies
	- neposílá se automaticky na server
### Bezpečnostní problémy
**Session hijacking** - útočník ukradne session ID a vydává se za uživatele
**CSRF (Cross-Site Request Forgery)** - útočník přiměje prohlížeč uživatele poslat request na cizí stránku (prohlížeč automaticky přiloží cookies)
**XSS (Cross-Site Scripting)** - útočník vloží JS kód do stránky přes vstup uživatele, který pak ukradne cookies nebo manipuluje stránkou
