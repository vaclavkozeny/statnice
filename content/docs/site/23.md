Základní principy WWW, HTTP, HTML. Jazyk (X)HTML, jeho charakteristika, možnosti a omezení
## WWW
- systém prohlížení, ukládání a odkazování dokumentů nacházejících se na internetu
- vznik 1989
### HTTP obecně
- **Hypertext Transfer Protocol**
- **Request/Response model**
	- funguje na principu dotaz/odpověď
	- klient pošle request, server vrátí response
#### Metody
- `GET` - získání zdroje, bez těla, idempotentní
- `POST` - odeslání dat na server (formulář, přihlášení)
- `PUT` - nahrazení zdroje
- `DELETE` - smazání zdroje
- `HEAD` - jako GET ale bez těla odpovědi (jen hlavičky)
- `OPTIONS` - zjištění co server podporuje
#### Kódy
- `2xx` – úspěch (200 OK, 201 Created)
- `3xx` – přesměrování (301 Moved Permanently, 302 Found)
- `4xx` – chyba klienta (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found)
- `5xx` – chyba serveru (500 Internal Server Error, 503 Service Unavailable)
#### **HTTP hlavičky**
- metadata přenášená s requestem/response:

- `Content-Type` – typ obsahu (text/html, application/json)
- `Authorization` – přihlašovací údaje
- `Cookie` / `Set-Cookie` – správa cookies
- `Cache-Control` – instrukce pro cachování
#### State
- **Cookies** – malé soubory uložené v prohlížeči, posílají se s každým requestem
- **Session** – stav uložen na serveru, klient má jen session ID (v cookie)
- **JWT token** – stav zakódován přímo v tokenu, server nic neukládá
### HTTP/1.1
- port 80
- textový přenos dat
- stateless protokol
- stav si uchovává klient
- výhody: robustní, snadnější implementace 
- nevýhody: větší režie, některé služby vyžadují stavové informace

### HTTPS
- HTTP over TLS
- port 443
- šifruje komunikaci
- ověřuje autenticitu serveru
### HTTP/2.0
- binární přenos dat
- streamy
	- klient navazuje jedno TCP spojení
- komprimace odopovědi
### QUIC
- Quick UDP Internet Connection
- L4 protokol
- UDP, port 443
- rychlejší navázání spojení
### HTTP/3.0
- podobné 2.0 ale s využitím QUIC
## HTML
- **Hypertext Markup Language**
- standardizovaný markup language
- definuje content a strukturu webů
- skupina značek (tagů)
- **Sémantické značky** 
	- HTML5 přineslo značky které nesou význam
	- `<header>`, `<footer>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`
```html
<!DOCTYPE html>
<html lang="cs">
  <head>
    <meta charset="UTF-8">
    <title>Název stránky</title>
  </head>
  <body>
    <h1>Nadpis</h1>
    <p>Odstavec</p>
  </body>
</html>
```
### Struktura webu
- HTML – struktura a obsah
- CSS – vzhled a layout
- JavaScript (ECMAScript) – chování a interaktivita
#### XHTML 
- přísnější verze HTML založená na XML
- všechny tagy musí být uzavřeny (`<br />` místo `<br>`)
- atributy musí být v uvozovkách
- tagy musí být malými písmeny
### Omezení
- samo o sobě je statické – bez JS žádná interaktivita
- vzhled bez CSS je primitivní
- neposkytuje žádnou logiku ani výpočty
- není programovací jazyk – je to značkovací jazyk
