Normalizace, normální formy, funkční závislosti, aktualizační anomálie

## Normalizace
### Funkční závislosti
- popisují vztahy mezi atributy v relaci
- říká, že hodnota jednoho atributu (nebo skupiny atributů) jednoznačně určuje hodnotu jiného atributu.
- **A -> B** - hodnota atributu A jednoznačně určuje hodnotu atributu B
- **identifikace atributů** které jsou **redundantní** nebo nekonzistentní
- základ pro normalizaci databáze
### Normalizace
- **minimalizovat redundanci dat**
- **zajistit konzistenci a integritu dat**
## Normy
### 0. NF
- alespoň jeden atribut obsahuje více než jednu hodnotu
### 1. NF
- každý atribut obsahuje atomické hodnoty
- každý záznam obsahuje unikátní identifikátor (primární klíč)

<hr>

Před

| student | kurzy        |
| ------- | ------------ |
| Jan     | MAT,FYZ,CHEM |
| Eva     | BIOL,FYZ     |

------------------------------------
Po

| student | kurz |
| ------- | ---- |
| Jan     | MAT  |
| Jan     | FYZ  |
| Jan     | CHEM |
| Eva     | BIOL |
| Eva     | FYZ  |

### 2. NF
- 1. NF
- všechny neklíčové atributy závisí na celém (složeném) klíči
- neexistuje FZ: část klíče -> neklíčový atribut

<hr>

Před

| Student | Kurz | Učitel |
| ------- | ---- | ------ |
| Jan     | MAT  | Novák  |
| Jan     | FYZ  | Novák  |
| Eva     | BIOL | Dvořák |

-------------------------------------------
Po

| Student | Kurz |
| ------- | ---- |
| Jan     | MAT  |
| Jan     | FYZ  |
| Eva     | BIOL |

| Kurz | Učitel |
| ---- | ------ |
| MAT  | Novák  |
| FYZ  | Novák  |
| BIOL | Dvořák |

### 3. NF
- 2. NF
- všechny neklíčové atributy jsou navzájem nezávislé
- žádný neklíčový atribut není tranzitivně závislý na klíči
- neexistuje FZ: neklíčový atribut -> neklíčový atribut

<hr>

Před

| ID  | Jmeno | Prijmeni | Pozice  | Plat  |
| --- | ----- | -------- | ------- | ----- |
| 66  | Jane  | Smith    | Účetní  | 31000 |
| 67  | John  | Doe      | Údržbář | 28000 |
| 68  | Jan   | Novák    | Účetní  | 31000 |


<hr>

Po

| ID  | Jmeno | Prijmeni | Pozice (FK) |
| --- | ----- | -------- | ----------- |
| 66  | Jane  | Smith    | Účetní      |
| 67  | John  | Doe      | Údržbář     |
| 68  | Jan   | Novák    | Účetní      |

| Pozice (PK) | Plat  |
| ----------- | ----- |
| Účetní      | 31000 |
| Údržbář     | 28000 |

### BCNF
- pro každou funkční závislost **A → B** musí platit, že **A je nadklíč** (superklíč) – tedy A jednoznačně identifikuje celý řádek
- v praxi se pak zůstává u 3. NF a závislosti se hlídají jinak (ap. logika, triggery)

## Aktualizační anomálie

- **Insertion anomaly** 
	- nemůžeš vložit data, aniž bys měl i jiná data
- **Update anomaly** 
	- stejná informace je uložena na více místech, a při změně musíš aktualizovat všechny výskyty
	- pokud změníš plat účetního na jednom řádku a zapomeneš na ostatní, data jsou nekonzistentní
- **Deletion anomaly** 
	- smazáním záznamu ztratíš i jinou informaci
